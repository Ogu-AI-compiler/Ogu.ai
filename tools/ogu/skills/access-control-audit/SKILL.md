# Access control audit

## When to Use

Activate this skill when:
- Working on access control audit tasks
- Reviewing or improving existing access control audit implementations
- Troubleshooting issues related to access control audit
- Setting up or configuring access control audit from scratch

## Workflow

1. Define threat model and attack surface scope
2. Assess current security posture and controls
3. Identify vulnerabilities using manual and automated methods
4. Prioritize findings by severity (CVSS score or business impact)
5. Develop and implement mitigations or compensating controls
6. Verify fixes with automated tests and manual validation
7. Document findings, evidence, and remediation steps
8. Map findings to compliance requirements if applicable
9. Schedule ongoing monitoring and review cadence

## Quality Bar

- No critical or high severity vulnerabilities left unmitigated
- All findings documented with evidence and CVSS scores
- Compliance requirements mapped and addressed
- Automated security scans configured in CI pipeline
- Remediation timeline agreed with stakeholders
