# Post processing

## When to Use

Activate this skill when:
- Working on post processing tasks
- Reviewing or improving existing post processing implementations
- Troubleshooting issues related to post processing
- Setting up or configuring post processing from scratch

## Workflow

1. Define gameplay requirements and performance targets (frame budget)
2. Design system architecture (entity-component, game loop, state machine)
3. Implement core mechanic or system
4. Profile performance against frame budget targets
5. Optimize hot paths (CPU, GPU, memory allocation)
6. Add debug visualization and tooling
7. Test across target platforms and hardware tiers
8. Document system design and performance characteristics

## Quality Bar

- Meets frame rate targets on all supported platforms
- No memory leaks or excessive allocations in hot paths
- Tested across hardware tiers
- System is deterministic where required (multiplayer)
