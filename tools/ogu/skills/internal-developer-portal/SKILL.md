# Internal developer portal

## When to Use

Activate this skill when:
- Working on internal developer portal tasks
- Reviewing or improving existing internal developer portal implementations
- Troubleshooting issues related to internal developer portal
- Setting up or configuring internal developer portal from scratch

## Workflow

1. Assess current state, constraints, and requirements
2. Design infrastructure or pipeline architecture
3. Implement with Infrastructure as Code (IaC) where possible
4. Add monitoring, alerting, and observability hooks
5. Test in a staging environment before production
6. Document runbooks and operational procedures
7. Review for security, cost efficiency, and reliability
8. Plan rollout and tested rollback strategy
9. Validate SLO/SLA requirements are met

## Quality Bar

- Infrastructure is reproducible via IaC (Terraform/Helm/etc.)
- Monitoring and alerts configured for all critical paths
- Runbooks documented and reviewed
- Rollback procedure tested and documented
- Cost baseline established with budget alerts
