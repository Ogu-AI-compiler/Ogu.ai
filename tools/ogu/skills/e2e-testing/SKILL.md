# E2e testing

## When to Use

Activate this skill when:
- Working on e2e testing tasks
- Reviewing or improving existing e2e testing implementations
- Troubleshooting issues related to e2e testing
- Setting up or configuring e2e testing from scratch

## Workflow

1. Understand requirements and define test scope
2. Design test strategy (unit / integration / e2e / exploratory)
3. Write test cases covering happy path, edge cases, and error conditions
4. Implement automated tests with clear naming and documentation
5. Execute test suite and analyze results
6. Triage failures: flaky tests vs. real bugs
7. File bugs with reproduction steps, expected vs. actual behavior
8. Verify bug fixes pass all relevant tests
9. Report quality metrics and coverage trends

## Quality Bar

- Test coverage meets target threshold
- All automated tests are deterministic (no flakiness)
- Bugs documented with clear reproduction steps
- Quality gate criteria satisfied for release
- Test suite runs in acceptable time
