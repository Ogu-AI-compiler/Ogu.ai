# Vertical feature delivery

## When to Use

Activate this skill when:
- Working on vertical feature delivery tasks
- Reviewing or improving existing vertical feature delivery implementations
- Troubleshooting issues related to vertical feature delivery
- Setting up or configuring vertical feature delivery from scratch

## Workflow

1. Define problem statement, success metrics, and evaluation protocol
2. Collect, validate, and document training and evaluation datasets
3. Select model architecture or base model for fine-tuning
4. Implement training/fine-tuning pipeline with experiment tracking
5. Evaluate model against baseline and target metrics
6. Optimize for inference performance (latency, throughput, cost)
7. Set up model monitoring and drift detection in production
8. Write model card (training data, limitations, metrics, intended use)
9. Version model artifacts and document reproducibility steps

## Quality Bar

- Metrics meet or exceed target thresholds
- Evaluation dataset is representative and unbiased
- Model card documented with limitations
- Inference latency meets SLO
- Experiment is reproducible from logged artifacts
