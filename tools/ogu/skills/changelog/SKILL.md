# Changelog

## When to Use

Activate this skill when:
- Working on changelog tasks
- Reviewing or improving existing changelog implementations
- Troubleshooting issues related to changelog
- Setting up or configuring changelog from scratch

## Workflow

1. Identify target audience and documentation goals
2. Audit existing documentation for gaps and inaccuracies
3. Gather technical details from code, SMEs, and product specs
4. Structure content with clear information architecture
5. Write accurate, concise, and example-rich content
6. Add working code samples and usage examples
7. Technical review: verify accuracy with engineering
8. Editorial review: clarity, readability, consistency
9. Publish, version, and set up a maintenance process

## Quality Bar

- Content is technically accurate and up-to-date
- Code examples are tested and working
- Clear and appropriate for target audience
- Internal links and external references valid
- Versioned alongside product releases
