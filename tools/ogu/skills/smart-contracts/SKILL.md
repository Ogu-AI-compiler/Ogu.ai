# Smart contracts

## When to Use

Activate this skill when:
- Working on smart contracts tasks
- Reviewing or improving existing smart contracts implementations
- Troubleshooting issues related to smart contracts
- Setting up or configuring smart contracts from scratch

## Workflow

1. Define on-chain vs off-chain logic boundaries
2. Design contract architecture with upgradeability in mind
3. Implement with gas optimization patterns from the start
4. Audit for common vulnerabilities (reentrancy, overflow, access control)
5. Write comprehensive tests (unit, invariant, fuzzing)
6. Deploy to testnet and verify behavior
7. Formal verification for critical paths where feasible
8. Document ABI, events, and administrative functions

## Quality Bar

- No known vulnerability patterns present
- Gas costs benchmarked and optimized
- Tests include adversarial scenarios
- Contract verified on block explorer
