# Analytics review

## When to Use

Activate this skill when:
- Working on analytics review tasks
- Reviewing or improving existing analytics review implementations
- Troubleshooting issues related to analytics review
- Setting up or configuring analytics review from scratch

## Workflow

1. Understand data sources, schemas, and freshness requirements
2. Design data model or pipeline architecture
3. Implement ingestion and transformation logic
4. Add data quality checks (nulls, duplicates, schema drift)
5. Implement error handling, retry logic, and dead-letter queues
6. Add monitoring, alerting, and data lineage tracking
7. Write tests for transformations (unit + end-to-end)
8. Document data definitions, lineage, and SLAs
9. Validate output against business requirements

## Quality Bar

- Data quality checks pass (completeness, consistency, accuracy)
- Pipeline is idempotent and resumable on failure
- Lineage tracked and documented
- Performance meets data freshness SLA
- Schema changes are backwards compatible or versioned
