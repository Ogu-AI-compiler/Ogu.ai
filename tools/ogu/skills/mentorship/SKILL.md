# Mentorship

## When to Use
- Onboarding a new engineer to the team and codebase
- Providing structured career guidance during 1:1s
- Developing a junior engineer's technical skills

## Workflow
1. Start with goals: understand what the mentee wants to learn and achieve
2. Create a learning plan with milestones and checkpoints
3. Use Socratic method: ask guiding questions before giving answers
4. Provide specific, actionable feedback on code reviews and designs
5. Track progress: celebrate improvements, adjust plan when goals change

## Quality Bar
- Mentee has a clear growth plan with measurable outcomes
- Feedback is specific and tied to observable behaviors
- Mentee is challenged but not overwhelmed — stretch assignments with support
- Mentorship is documented so it can be recognized in performance reviews
