# Responsive implementation

## When to Use

Activate this skill when:
- Working on responsive implementation tasks
- Reviewing or improving existing responsive implementation implementations
- Troubleshooting issues related to responsive implementation
- Setting up or configuring responsive implementation from scratch

## Workflow

1. Review design specs, wireframes, or requirements
2. Set up component structure and file organization
3. Implement core functionality with proper state management
4. Apply styling and responsive design patterns
5. Handle edge cases, loading states, and error states
6. Write unit and component tests
7. Review for accessibility (WCAG 2.1 AA minimum)
8. Optimize for performance (bundle size, render cycles, LCP/CLS)
9. Document component API and usage examples

## Quality Bar

- All tests pass (unit, component, e2e)
- Meets accessibility standards (WCAG 2.1 AA)
- Responsive across target breakpoints
- No console errors in production mode
- Core Web Vitals within target thresholds
